# 7 주차 “현실 퀘스트” 게임 기획

목표 : 자신만의 게임 기획하고 계획 세우기

## 게임 제목 : 현실 퀘스트

## 게임 소개

- 제한 시간 안에 요구한 물건을 현실에서 찾아서 문제를 해결하는 게임
- 스토리 : 플레이어는 현실 세계에 실수로 떨어진 마법사로 다시 원래 세계로 돌아가기 위해 그의 스승님과 대화를 하면서 현실에서 특정 물체를 찾아야 한다. 만약에 다 찾지 못하면 이 세계에 홀로 남게 된다.
- 핵심 컨셉 : 플레이어의 방이 게임의 무대가 되어 화면에서 보여주는 퀘스트에 필요한 '실제 사물'을 웹캠에 보여줘서 해결하는, 증강현실(AR) 같은 게임
- 핵심 AI 모델: 객체 탐지 (Object Detection) 모델 : facebook/detr-resnet-50
    - 역할: 웹캠 영상에서 플레이어가 보여주는 물건이 무엇인지(예: '컵', '책', '스마트폰') 실시간으로
    인식하고 판단하는 역할
- 게임 흐름 구상:
    1. 플레이어에게 화면에서 스토리 배경을 설명 및 퀘스트 시작 
    2. 본격적인 게임 시작 - NPC가 플레이어에게 자기 방에서 '스마트폰'을 찾아달라고 한다. 
    3. 플레이어가 스마트폰을 가져와서 웹캠에 보여줌 
    4. AI가 '스마트폰'을 인식
    5. 해당 퀘스트를 완료 처리 후 다음 퀘스트 제출
    6. 제한 시간 안에 10가지 물체를 다 찾으면 성공 버전 스토리  / 못 찾으면 실패 버전 스토리 보여줌
- 중요한 부분 : 게임 속 세계와 플레이어의 현실 공간을 직접적으로 연결하여 플레이어는 단순히 키보드를 누르는 것을 넘어, 직접 몸을 움직여 자기 방 안의 사물과 상호작용하며 게임을 플레이하도록 구상

## 선정 이유

- 요즘 AI 활용이 트렌드 이며 파이썬에서 AI 모델을 쉽게 이용 가능한 것 같아 이를 활용한 게임을 제작하고 싶어 위 게임을 기획하게 되었습니다.
- AI 중에서 원래는 LLM 모델을 사용할까 했지만 테스트 결과 잘 나오지 않고 완벽한 통제가 어려워서 해당 모델보다 비교적 간단한 물체 인식 모델을 사용하게 되었습니다.
