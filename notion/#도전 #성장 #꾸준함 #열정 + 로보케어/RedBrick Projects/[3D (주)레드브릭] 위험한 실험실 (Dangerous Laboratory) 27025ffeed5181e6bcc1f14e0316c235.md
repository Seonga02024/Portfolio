- **다익스트라 알고리즘를 사용한 이유 :**

다익스트라 알고리즘은 그래프에서 한 정점에서 다른 모든 정점까지의 최단 경로를 찾는 알고리즘입니다. 이 알고리즘은 간단하고 직관적인 구현이 가능하며, 음수 가중치가 없는 그래프에서 효율적으로 작동합니다. 원래 다익스트라 알고리즘과 A* 알고리즘은 고려했으나 아래와 같은 이유를 생각해 선택하게 되었습니다.

1. 간단하고 직관적인 구현 - 우선순위 큐를 사용하여 노드를 선택하고 갱신하는 방식으로 동작하기 때문에 구현이 비교적 간단하다고 생각했습니다.
2. 맵 자체가 간단하고 모든 가중치가 음수가 없고 일정 했기 때문에 다익스트라가 유리하고 생각

- **다익스트라 알고리즘 구현 코드**
    
    ```jsx
    // 그래프 정의
    const graph = {
        0: [{node: 8, weight: 1}, {node: 6, weight: 1}, {node: 5, weight: 1}, {node: 7, weight: 1}],
        1: [{node: 8, weight: 1}, {node: 5, weight: 1}],
        2: [{node: 5, weight: 1}, {node: 6, weight: 1}],
        3: [{node: 7, weight: 1}, {node: 6, weight: 1}],
        4: [{node: 7, weight: 1}, {node: 8, weight: 1}],
        5: [{node: 1, weight: 1}, {node: 0, weight: 1}, {node: 2, weight: 1}],
        6: [{node: 0, weight: 1}, {node: 3, weight: 1}, {node: 2, weight: 1}],
        7: [{node: 4, weight: 1}, {node: 3, weight: 1}, {node: 0, weight: 1}],
        8: [{node: 1, weight: 1}, {node: 4, weight: 1}, {node: 0, weight: 1}],
    };
    
    // 다익스트라 알고리즘 함수
    function dijkstra(graph, start, end) {
        const distances = {}; // 시작 지점으로부터 각 지점 별 몇 번 거쳐왔는지 저장하고 비교하는 배열 
        const visited = {}; // 도착 여부 배열
        const queue = []; // 현재 검사해야하는 지점을 담는 배열
        const path = {}; // 시작 지점에서 끝 지점으로 향하는 각 지점 별 연결된 최단 지점
    
        distances[start] = 0; // 시작 지점
        queue.push({node: start, weight: 0});
    
        while (queue.length > 0) {
    		    // 가중치 낮은 순으로 검사 -> 한번 검사 노드는 두 번 검사하지 않아서
            queue.sort((a, b) => a.weight - b.weight);  
            const {node, weight} = queue.shift();
    
            if (visited[node]) continue; // 두 번 검사면 패스
            visited[node] = true; // 검사 했다 완료 표시
    
            for (const neighbor of graph[node]) { // 갈 수 있는 지점 다 돌기
                const newWeight = weight + neighbor.weight; // 가중치 계산
                // 기존 가중치 보다 낮거나 아직 정의 되어 있지 않을 경우
                if (distances[neighbor.node] === undefined || newWeight < distances[neighbor.node]) {
                    distances[neighbor.node] = newWeight; // 가중치 업데이트
                    path[neighbor.node] = node; // 경로 업데이트
                    queue.push({node: neighbor.node, weight: newWeight});
                }
            }
        }
        
        // 최단 경로 노드 출력
        let currentNode = end;
        const shortestPathNodes = [end];
        while (currentNode !== start) {
            currentNode = path[currentNode];
            shortestPathNodes.unshift(currentNode);
        }
    
        return shortestPathNodes;
    }
    ```

            const vz = (dz / distance) * this.speed;
        
            // 현재 위치 업데이트
            this.object.position.x += vx;
            this.object.position.z += vz;
            
            const angle = Math.atan2(this.destination.x - this.object.position.x, this.destination.z - this.object.position.z);
            this.object.rotation.y = angle - 90;
            this.object.body.needUpdate = true;
            
            // 역동감을 위한 귀신의 위아래 움직임 추가
            const range = 0.01; 
            const newY = this.object.position.y + range * Math.sin(Date.now() * 0.001);
            this.object.position.y = newY;
            this.object.body.needUpdate = true;
        }
        getRandomDestination() {
            let closestPoints = this.movePos.slice();
            let num = 0;
            closestPoints.forEach((point) => {
                const distance = point.position.distanceTo(PLAYER.position);
                point.distanceToPlayer = distance;
                point.index = num; 
                num++;
            });
            closestPoints.sort((a, b) => a.distanceToPlayer - b.distanceToPlayer);
            closestPoints.splice(4);
            let randomNum = closestPoints[Math.floor(Math.random() * closestPoints.length)].index;
    
            return randomNum;
        }
        moveTeleport(){
            this.isTeleport = true;
            
            let randomNumber = Math.floor(Math.random() * this.movePos.length);
            this.object.position.set(this.movePos[randomNumber].position.x, this.object.position.y, this.movePos[randomNumber].position.z);
            this.object.body.needUpdate = true;
            
            setTimeout(() => {
                this.isTeleport = false;
            }, 1000)
        }
    }

- 추가적인 설명
    
    Finite State Machines (상태 패턴 알고리즘) :
    
    - NPC의 상태를 정의하고, 각 상태에 따라 특정 행동을 수행하도록 구현.
    - NPC는 상태 전이를 통해 상태를 변경하고, 각 상태에 따라 다른 행동을 수행.
    
    ```jsx
    // NPC의 상태 정의
    const NPCState = {
        IDLE: 'IDLE',
        WANDER: 'WANDER',
        CHASE: 'CHASE'
    };
    
    // NPC 클래스
    class NPC {
        constructor() {
            this.state = NPCState.IDLE; // 초기 상태는 IDLE
        }
    
        // Finite State Machine 업데이트
        update() {
            switch (this.state) {
                case NPCState.IDLE:
                    console.log("NPC is idle");
                    // IDLE 상태에서 필요한 작업 수행
                    break;
                case NPCState.WANDER:
                    console.log("NPC is wandering");
                    // WANDER 상태에서 필요한 작업 수행
                    break;
                case NPCState.CHASE:
                    console.log("NPC is chasing");
                    // CHASE 상태에서 필요한 작업 수행
                    break;
                default:
                    console.log("Invalid state");
            }
        }
    
        // 상태 변경
        changeState(newState) {
            this.state = newState;
        }
    }
    
    // NPC 인스턴스 생성
    const npc = new NPC();
    
    // NPC 상태 변경 및 업데이트
    npc.changeState(NPCState.WANDER);
    npc.update();
    
    npc.changeState(NPCState.CHASE);
    npc.update();
    
    ```
