GAME TECH OVERVIEW

Silbot 미니게임을 떠받치는 기술

본인이 메인으로 담당한 부분을 다루기 전에, 세 게임 전반을 떠받치는 기술 결정과 공통 구조를 먼저 정리합니다. 팀이 함께 만든 토대 위에서 각자의 영역이 어떻게 나뉘었는지 보여주기 위함입니다.

A BaseManager · 공통 게임 라이프사이클

모든 미니게임 매니저가 상속하는 BaseManager : MonoBehaviourPunCallbacks. 게임 상태(튜토리얼·레벨선택·플레이·종료), Photon RPC 송수신, Socket.IO 연결, 게임 데이터 수집을 하나의 추상 계층으로 통일.

  • SocketIO3Connect(SocketProtocal) — 게임별 채널 자동 바인딩
  • 게임 상태 머신 — TUTORIAL → SELECT_LEVEL → DOING_GAME → FINSH_GAME
  • RPCSend/CountAnswer — 학생 간 진행 동기화 표준화

B Photon + Socket.IO 이중 네트워크

실시간 학생-간 동기화는 Photon PUN 의 RPC, 교사-학생 출제·제어는 Socket.IO 의 텍스트 프로토콜로 역할을 분리. 한 채널이 끊겨도 다른 채널은 살아남아 수업이 멈추지 않음.

  • Photon — 학생-학생, 점수·공격·콤보 등 빠른 동기화
  • Socket.IO — 교사 → 학생 명령 (SELECT_LEVEL^level, GAME_START, SHOW_HINT 등)
  • UnityMainThreadDispatcher 로 소켓 콜백을 메인 스레드에 안전하게 전달

C RobotManager · 실벗 발화/표정 통합

게임 로직과 로봇 제어를 완전히 분리. 게임 매니저는 "이런 상황이에요" 라고 신호만 보내면, RobotManager 가 TTS·표정·움직임을 알아서 트리거. 게임 코드에 로봇 SDK 의존성이 새지 않음.

  • TTS 발화 종료 콜백(RobotSentenceEnd)으로 게임 진행 동기화
  • 얼굴 테마 프로토콜 — 표정 키워드를 명령으로 전송
  • 볼륨·배터리·연결 상태 모니터링
  • HiBomi2 보조 로봇 연결 지원

D IntentSetting · 외부 환경 주입

안드로이드 Intent 로 앱 실행 시점에 게임 API URL·TTS·로봇 ID·프로젝트 설정을 받아오는 구조. 빌드 한 번으로 여러 학교/기관 환경에 동일한 APK 를 배포 가능.

  • Intent → IntentSetting → 전역 설정 캐시
  • 개발자용 설정 패널 — Socket URL, DB, TTS 모드 직접 변경
  • 로그인 조건 분기 처리 (게스트/계정/실벗 단독)
PER-GAME DEEP DIVE

본인 담당 세 게임의 핵심 구조

실벗 카탈로그 20여 종 중 본인이 메인으로 담당한 세 게임 — 3-11 Match3 · 5-06 Match · 5-07 Star — 각각의 핵심 클래스·로직을 정리하고, 그 중에서도 단독으로 책임진 부분을 MAIN 배지로 표시합니다. 세 게임 모두 팀과 공동 설계한 클래스 구조 위에서 구현되었습니다.

3-11 · MATCH3

3매치 퍼즐

드래그 스왑 → 3+ 매치 검출 → 제거·중력·연쇄. 4단계 그리드(7×6 ~ 11×9) · 폭탄 4종 · 힌트 시스템.

Match3Game · 매치 검출/연쇄

그리드 2D 배열 위에서 스왑 직후 가로·세로 3+ 매치를 스캔, 제거 후 위쪽 타일을 떨어뜨려 빈 칸을 채우고 새 매치가 없을 때까지 반복(체인 리액션). tilesToRemoveCache 로 GC 부담 최소화.

Tile · BombTile 다형 구조

Tile.CalculationRemoveRange() 가상 메서드를 BombTile(3×3) · HorizontalLine · VerticalLine · RainBomb 이 각각 오버라이드해 폭탄별 제거 범위를 자체적으로 계산. 새 폭탄 추가가 한 클래스로 끝나도록 설계.

힌트 · 보너스 코루틴

5초 입력 없을 때 가능한 스왑을 깜빡임으로 안내. 보너스 타임에는 DOTween 으로 타일 회전 효과. hintCoroutine, tileRemoveCoroutine 등 타이밍 코루틴 일체.

Match3Manager · 네트워크 어댑터

BaseManager 상속. 레벨·시간 설정 명령(SELECT_LEVEL^level)을 받아 게임 시작, 점수를 RPC 로 다른 학생에게 동기화. 팀과 공동 설계, 메인 작업.

Master 측 Match3Play

교사 PC 의 RCPlayManager2 상속 클래스. GOOD_JOB·ADD_TIME· SEND_EVENT(보너스) 등 명령 송신. 팀원 작업과 통신 프로토콜만 맞춤.

5-06 · MATCH

합이 10! 사과 잡기

드래그로 사과를 선택 → 합이 10이면 성공. 콤보 누적 시 상대 보드 공격. 가중치 기반 사과 숫자 분포.

MouseDragController · 드래그 선택

마우스/터치 다운→드래그→업 흐름에서 사과 위치·거리 검사. 인접 거리(200) 초과 시 점프 방지, 선택 사과를 LineRenderer 로 연결. 업 시점에 합 계산해 ANSWER(10) 여부 판정.

가중치 사과 스폰

1~9 숫자 사과를 그냥 랜덤이 아니라 weightedNumbers = [1,1,2,2,3,3,4,4,5,6,7,8,9] 가중치로 분포. 마지막 사과는 전체 합이 10의 배수가 되도록 보정. 풀이 가능한 보드 자동 보장.

콤보 → 공격 전환

레벨별 comboNum(5~10) 도달 시 OtherPlayerAttack() RPC 호출. 상대 학생 보드에 파티클 이펙트를 발생시키고 콤보 카운터를 리셋. 랭크 모드에서만 활성.

MatchManager · BaseManager 통합

Photon RPC 와 Socket.IO 채널 연결, 학생 수 동기화, 점수 정규화. 다른 미니게임과 동일 패턴으로 작성.

CollaborationManager (Master)

협력 모드에서 학생 전원의 점수를 합산해 단계별로 표시. 이번 라운드 점수 · 누적 점수 동시 추적. 팀원과 공동 작업.

CODE SAMPLE

한 눈에 보는 폭탄 타일 다형 구조

매치 게임의 폭탄 타일은 모두 같은 Tile 베이스를 상속받습니다. 매니저는 폭탄 종류를 알 필요 없이 CalculationRemoveRange() 만 호출하면 됩니다.

// Tile.cs — 베이스 클래스 

public class Tile : MonoBehaviour
{
    public int xIndex, yIndex;
    public int itemIndex;       // 0~4 일반 색, 5+ 폭탄/특수
    public bool isItem;

    // 폭탄별 제거 범위를 자체 계산하도록 위임
    public virtual List<Tile> CalculationRemoveRange(Tile[,] grid, Tile partner)
    {
        return new List<Tile>();
    }
}

// HorizontalLineBombTile.cs — 가로 한 줄 제거 

public class HorizontalLineBombTile : Tile
{
    public override List<Tile> CalculationRemoveRange(Tile[,] grid, Tile partner)
    {
        var result = new List<Tile>();
        int width = grid.GetLength(0);
        for (int x = 0; x < width; x++)
            if (grid[x, yIndex] != null) result.Add(grid[x, yIndex]);
        return result;
    }
}

// Match3Game.cs — 폭탄 종류를 모르고도 동일하게 처리 

// 스왑 직후 매치 검사에서 만난 폭탄을 같은 인터페이스로 처리
foreach (var tile in matchedTiles)
{
    if (!tile.isItem) continue;

    // 어떤 폭탄이든 한 메서드로 제거 범위를 받아옴
    var extra = tile.CalculationRemoveRange(grid, swappedPartner);
    tilesToRemoveCache.AddRange(extra);
}

// 캐시를 한 번에 비워 GC 부담 최소화
foreach (var t in tilesToRemoveCache) RemoveTile(t);
tilesToRemoveCache.Clear();

// BaseManager.cs — Socket.IO + Photon 이중 채널 진입점 

public abstract class BaseManager : MonoBehaviourPunCallbacks
{
    // 게임별 SocketProtocal 만 넘기면 채널 자동 바인딩
    protected void SocketIO3Connect(SocketProtocal proto) { ... }

    // 학생 전원에게 동시 RPC — Photon 방 전체 브로드캐스트
    protected void RPCSend(string method, params object[] args)
    {
        photonView.RPC(method, RpcTarget.All, args);
    }
}
