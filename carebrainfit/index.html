GAME TECH OVERVIEW

CareBrainFit 의 기술적 핵심

본인이 직접 작업한 영역으로 들어가기 전, 게임 전반을 떠받치는 기술 결정들을 먼저 정리합니다. 팀이 공통으로 만든 토대 위에서 각자의 영역이 어떻게 분담되었는지 보여주기 위함입니다.

A MediaPipe AI 신체 추적

컨트롤러 없이 게임을 즐길 수 있도록 MediaPipe Unity 패키지를 활용해 전신 자세·얼굴 키포인트를 실시간 추적. 데스크탑·안드로이드 양쪽에서 동작.

  • Pose Estimation — 전신 33개 키포인트
  • Face Detection — A4 회피 게임의 입력 수단
  • PoseManager 가 결과를 정규화·이벤트화하여 게임 로직과 분리

B 확장 가능한 아케이드 구조

새 아케이드 콘텐츠 추가 시 동일한 패턴(BasicArcadeView 상속 + ArcadeSubscriber 이벤트 구독)으로 일관되게 작성할 수 있는 베이스 계층 설계.

  • BasicArcadeView · BasicArcadeEvents · BasicArcadeManager 3-계층
  • OnSubscribed / OnStarted / OnFinished / OnPaused / OnRerun 훅
  • 현재 A1~A4 모두 동일 구조 위에서 동작

C 이벤트 기반 아키텍처

매니저 간 직접 참조 대신 정적 이벤트(GameEvents · SystemEvents · ArcadeEvents · PoseEvents · UIEvents) 로 결합도 낮춤. 한 매니저 교체가 다른 매니저에 영향 주지 않음.

  • 5종 정적 이벤트 클래스로 도메인 분리
  • DataManager · UI · 서버 매니저가 같은 이벤트 구독
  • 씬 전환 시 ResetEvents() 로 누수 방지

D 데이터 드리븐 콘텐츠 카탈로그

각 게임을 GameEntry ScriptableObject 로 정의하고 GameCatalog 가 일괄 관리. 새 콘텐츠 추가 시 코드 수정 없이 에셋 추가만으로 그리드 UI 자동 갱신.

  • GameEntry (id, 썸네일, 영상, 설명, 난이도)
  • ContentGridBuilder · ContentItemView 가 자동 렌더링
  • Remote Config 와 결합되어 경제 정보까지 외부에서 주입

서버 검증 화폐 시스템 (PLAYER_MONEY)

CODE SAMPLE

한 눈에 보는 화폐 흐름

클라이언트는 contentId 만 보내고, 서버가 Remote Config 를 참조하여 실제 차감액·지급액을 결정합니다. 클라이언트가 amount 를 위조해도 무시됩니다.

// Client (C#) — DataManager.cs 

// 게임 진입 시 차감 — 차감액은 서버가 RC 에서 결정
public async Task<bool> SpendMoneyAsync(string contentId)
{
    if (string.IsNullOrEmpty(contentId)) return false;

    var req = new SpendPlayerMoneyRequest { contentId = contentId };
    var res = await ServerEventManager.Instance.SpendPlayerMoneyAsync(req);

    if (res != null && res.success)
    {
        ApplyServerMoney(res.money);  // 캐시만 갱신
        return true;
    }
    UIEvents.OnSpendMoneyFailed?.Invoke();
    return false;
}

// Server (Cloud Code JS) — SpendPlayerMoney.js 

const economy = await loadEconomy(...);  // Remote Config 조회
const entry = economy[contentId];
if (!entry) return build(false, ..., "UNKNOWN_CONTENT");

const cost = Number(entry.cost);
if (cost <= 0) return build(false, ..., "NOT_SPENDABLE");

// WriteLock 재시도로 동시성 처리
const current = await readMoney(...);
if (current.money < cost) return build(false, ..., "INSUFFICIENT_FUNDS");

await cloudSaveApi.setItem(projectId, playerId, {
    key: PLAYER_MONEY_KEY,
    value: current.money - cost,
    writeLock: current.writeLock,
});
return build(true, playerId, contentId, current.money - cost, cost, "OK");
