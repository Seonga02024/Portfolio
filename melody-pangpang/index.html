직접 설계 · 구현한 4가지 핵심 시스템

디자이너 1명과 협업하여 2주 만에 출시까지 마친 1인 개발. 음악 동기화, 노드 판정 최적화, GUI 이펙트, 점수·콤보·HP 게임 시스템을 처음부터 끝까지 설계했습니다.

AREA 01 · MUSIC SYNC

음악 동기화 시스템

사용자마다 다른 PC 사양에서도 박자가 어긋나지 않도록, 노래 재생 시간 기점으로 박자를 계산하는 동기화 파이프라인 설계.

  • music.context.currentTime - music._startedAt 기점 박자 계산
  • 180 BPM 기준 비트 타임 배열에 leadTime 더해 미리 노드 생성
  • 게임 시작 시점이 아닌 노래 재생 시간 기준 → 사양 차이 흡수
  • 종료 시 음악 → 타이틀 음악 자연 교체
AREA 02 · JUDGEMENT OPT

노드 판정 부하 최적화

매 클릭마다 전체 노드를 검사하면 과부하. 고유 인덱스 기반으로 검사 범위를 8개로 한정하여 입력 응답성을 일정하게 유지.

  • 각 노드에 currentNodeNum 고유 인덱스 부여
  • lastClickBox ±4 범위만 검사 → 검사 8개로 한정
  • X 좌표 비교로 Left/Up/Down/Right 기둥 빠르게 식별
  • 거리별 Perfect (≤0.2) · Cool (≤0.5) · Miss (≤1) 판정
AREA 03 · GUI / FX

GUI 이펙트 & 시각 피드백

리듬 게임 레퍼런스에서 검증된 시각적 쾌감을 그대로 살리면서, TWEEN 이징으로 판정 UI의 등장 모션과 콤보 표시를 자연스럽게.

  • TWEEN.Easing.Quartic.InOut 으로 판정 UI 스케일 애니메이션
  • 키 누름 시 배경 빛 기둥, 노드 진입 시 라이트 효과
  • 최초 Perfect부터 콤보 GUI 시작 · Miss 시 초기화 연출
  • Miss 시 카메라 살짝 흔들림으로 다중 피드백
AREA 04 · GAME SYSTEMS

점수 · 콤보 · HP 시스템

짧은 한 곡(20~30초) 안에서 도전 욕구를 끌어내는 점수/HP 룰 설계. 랭킹 경쟁이 자연스럽게 발생하는 변동성 곡선.

  • Perfect 20점 / Cool 10점 / Miss 0점 기본 점수
  • 20콤보부터 1.5배 · 50콤보부터 2배 콤보 보너스
  • HP 100 시작 · Miss 시 -10 · 0 도달 시 게임 종료
  • 고점수 = 정확도 + 콤보 유지 + 풀 곡 클리어 삼박자

src/game.js · 노드 생성 + Miss 판정 JavaScript 
const boxSpeed = 5;             // 박스 하락 속도
const leadTime = 2;             // 비트 타이밍보다 미리 생성할 시간 (초)
const trigger_y = 0;            // 판정 라인 Y 값
const startBeatTimes = 3.37;
const trigger_x = [-2, -1.2, -0.4, 0.4];  // 4개 기둥 X 좌표
const timeAdjustment = 0.5;

function calBoxPosition(time, speed, targetY, trigger_x, index) {
    // 시작 위치를 계산: initialY = targetY + (speed * time)
    const initialY = targetY + (speed * time);
    const initialX = trigger_x[index % 4];
    const initialZ = 66;
    return {x: initialX, y: initialY, z: initialZ};
}

function Update(dt) {
    if (music.isPlaying) {
        // 노래 재생 시간 기점으로 박자 계산 (게임 시작 기점 X)
        const playTime = music.context.currentTime - music._startedAt;

        beatTimes.forEach((beatTime, index) => {
            // 현재 재생 시간이 beatTime - leadTime에 도달했을 때 박스 생성
            if (playTime >= (beatTime + timeAdjustment - leadTime) && !downBoxes[index]) {
                let newBoxes = [];
                beatNodePattern[currentNodePatternNum].forEach((num, index) => {
                    if(num === 1){
                        let newBox = /* index 별 left/up/down/right 노드 클론 */;
                        const initialY = calBoxPosition(playTime, boxSpeed, trigger_y, trigger_x, index).y;
                        newBox.position.set(trigger_x[index % 4], initialY, 66);
                        newBox.isClick = false;
                        newBox.index = currentNodeNum;     // 고유 인덱스 부여
                        WORLD.add(newBox);
                        newBoxes.push(newBox);
                        beatBoxes.push(newBox);
                        currentNodeNum++;
                    }
                });
                downBoxes[index] = newBoxes;
                currentNodePatternNum++;
            }
        });

        // 박스 하락 + Miss 판정
        downBoxes.forEach((boxes) => {
            if (boxes) {
                boxes.forEach((box) => {
                    box.position.y -= boxSpeed * dt;
                    if (box.position.y <= trigger_y && box.isClick === false) {
                        gameGUI.clickNode(false, false, true);  // Miss 표시
                        box.isClick = true;
                        currentCombo = 0;
                        missDamge();
                    }
                });
            }
        });
    }
}
src/game.js · CheckDownBoxes (인덱스 기반 부하 최적화) JavaScript 
function CheckDownBoxes(keyNum){
    let compareBox = null;
    let compareX = 0;
    let findNearBox = null;
    let nearDistance = 10000000;

    switch(keyNum){
        case "Left":  compareBox = leftBox.position;  compareX = trigger_x[0]; break;
        case "Up":    compareBox = upBox.position;    compareX = trigger_x[1]; break;
        case "Down":  compareBox = downBox.position;  compareX = trigger_x[2]; break;
        case "Right": compareBox = rightBox.position; compareX = trigger_x[3]; break;
    }

    // 마지막 친 노드 기준 ±4개 노드만 검사 (전체 검사 X)
    for(let i=-4; i<4; i++){
        if(lastClickBox + i >= 0 && beatBoxes[lastClickBox + i]){
            let box = beatBoxes[lastClickBox + i];
            // X 좌표로 같은 기둥인지 확인
            if(box.position.x === compareX && box.isClick === false){
                let distanceY = Math.abs(box.position.y - compareBox.y);
                if(distanceY < nearDistance){
                    findNearBox = box;
                    nearDistance = distanceY;
                }
            }
        }
    }

    if(findNearBox != null){
        findNearBox.isClick = true;
        lastClickBox = findNearBox.index;

        if(nearDistance <= 0.2){          // Perfect
            currentCombo++;
            let comboScore = 20;
            if(currentCombo >= 50) comboScore *= 2;
            else if(currentCombo >= 20) comboScore *= 1.5;
            gameGUI.clickNode(true, false, false);
            gameGUI.settingScoreUI(comboScore);
        } else if(nearDistance <= 0.5){   // Cool
            currentCombo++;
            let comboScore = 10;
            if(currentCombo >= 50) comboScore *= 2;
            else if(currentCombo >= 20) comboScore *= 1.5;
            gameGUI.clickNode(false, true, false);
            gameGUI.settingScoreUI(comboScore);
        } else if(nearDistance <= 1){     // Miss
            gameGUI.clickNode(false, false, true);
            currentCombo = 0;
            missDamge();
        }
    }
}
src/gameGUI.js · clickNode (TWEEN.js 판정 UI 이펙트) JavaScript 
clickNode(isPerfect, isCool, isMiss){
    this.perfectUI.hide();
    this.coolUI.hide();
    this.missUI.hide();

    let currentJudeUI = null;
    if(isPerfect)   { this.perfectUI.show(); currentJudeUI = this.perfectUI; }
    else if(isCool) { this.coolUI.show();    currentJudeUI = this.coolUI; }
    else if(isMiss) { this.missUI.show();    currentJudeUI = this.missUI; }

    // 판정 UI 가 커지면서 등장하는 이펙트
    const start = { x: 20 };
    const end = { x: 25 };

    new TWEEN.Tween(start)
        .to(end, 200)
        .onUpdate((obj) => { currentJudeUI.size.x.value = start.x; })
        .easing(TWEEN.Easing.Quartic.InOut)
        .start();
}
// problem_solving

Real problems, shipped solutions

1인 개발 과정에서 실제로 마주친 4가지 문제와 해결 과정. 리듬 게임의 본질인 "박자 정확도"와 "체감 응답성"을 어떻게 잡아냈는지.

Beat Sync
컴퓨터 사양마다 노래 시작 타이밍이 달라 박자가 어긋남
solution
게임 시작 시점이 아닌 music.context.currentTime - music._startedAt 을 기점으로 박자를 계산. 어떤 환경에서도 박자가 어긋나지 않도록 시계 기준을 음악에 고정.
Input Latency
매 입력마다 모든 노드를 검사하면 입력 응답이 일정하지 않음
solution
노드에 고유 인덱스를 부여하고, 마지막으로 친 노드 기준 ±4 만 순회. 검사 범위를 8개로 한정하여 입력 → 판정 지연을 일정하게 유지.
Visual Feedback
판정 UI를 그냥 띄우기만 하면 리듬 게임 특유의 쾌감이 부족
solution
TWEEN.Easing.Quartic.InOut 으로 판정 UI가 커지며 등장하는 이펙트, Miss 시 카메라 살짝 흔들림과 콤보 GUI 사라짐으로 다중 피드백을 겹쳤어요.
Failure UX
Miss 시 점수만 차감되면 실패의 무게감이 없음
solution
점수 0점 + HP -10 + 콤보 시각적 초기화 + 카메라 흔들림을 동시에. 한 번의 Miss에서 시각/청각/수치 피드백이 겹치며 다음 노드에 집중하게 만듭니다.
